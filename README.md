# NPO Orphan Care System

Lightweight orphan family management system for non-profit organizations.
Two complementary subsystems: a **workbook scanner** and a **desktop app**.

## Repository layout

```
npo-orphan-care-sys/
├── main.ods              # LibreOffice Calc workbook (master data + forms)
├── convert.py            # Scanner: analyzes main.ods, exports JSON/CSV
├── mini_db/              # Desktop app: Tkinter GUI with multi-backend DB
│
├── documents/            # File store (mirrors DB table names)
├── exports/              # Generated by convert.py (schema, forms, relationships)
│
├── requirements.txt      # Python dependencies
├── setup.bat / .ps1      # One-click Windows setup (Chocolatey + venv)
└── run.bat / .ps1        # Launch scanner
```

## Two subsystems

### 1. Workbook Scanner (`convert.py`)

Reads `main.ods` (25 sheets) via pandas, then:

- **Form detection** — Finds 9 data-entry forms on `input_dashboards` sheet
  by scanning for header cells matching "Register/Add/Patch * Form".
  Walks downward to collect field labels and coordinates.
- **Data extraction** — Reads each entity sheet (Persons, Families, Orphans,
  etc.) as a table with header-row detection and record export to JSON.
- **Relationship mapping** — Detects 27 foreign-key relationships across
  the schema by matching `_id` column suffixes against known table aliases.
- **Output** — Writes to `exports/`:
  - `schema.json` — table schemas (columns, types)
  - `forms.json` — form structure (headers, fields, coordinates)
  - `relationships.json` — FK graph
  - `lookups.json` — small reference tables (Cities, Banks, etc.)
  - `data/*.json` — raw record exports

### 2. Desktop App (`mini_db/`)

Tkinter GUI with three user roles (Manager / Volunteer / Staff).
See `mini_db/README.md` for full details.

### 3. LibreOffice Macros (`mini_db/kahatayn_macros.py`)

16 Python macros (CRUD + search) deployed to LibreOffice's script directory
and assigned to buttons on `input_dashboards`. Requires the **APSO extension**
to bridge Python and LibreOffice.

## Quick start

### Windows (recommended)

```powershell
# Double-click, or run:
.\setup.ps1

# Then:
.\run.ps1
```

`setup.ps1` uses Chocolatey to install Python + LibreOffice, creates a
virtual environment, and installs dependencies.

### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python3 convert.py
```

### After setup

| Task | Command |
|---|---|
| Scan workbook | `python convert.py` |
| Launch desktop app | `cd mini_db && python main.py` |
| Install APSO extension | https://extensions.libreoffice.org/…/apso |
| Deploy macros | `copy mini_db\kahatayn_macros.py "%APPDATA%\LibreOffice\4\user\Scripts\python\"` |
| Run tests | `cd mini_db && python demo_new_features.py` |

## Database schema

23 tables across 25 ODS sheets. Core entities:

```
Persons ──┬── Employees      Families ──┬── Family_member
          ├── Reps                       ├── Orphans
          ├── Donors                     ├── Assessments
          ├── Bank_account               └── Sponsorships
          └── Contact
```

Reference/lookup tables: City, Schools, Banks, Bank Branches, deposit type,
document type, spons_type, criteria. Activity log: Audit.

See `exports/schema.json` and `exports/relationships.json` for the full
structure (generated by `python convert.py`).

## Requirements

- Python 3.10+
- LibreOffice (for macro-based data entry)
- APSO extension (for Python macro support in LibreOffice)

Python packages (installed by setup scripts):

| Package | Used by |
|---|---|
| pandas, odfpy, openpyxl | convert.py + mini_db |
| SQLAlchemy | mini_db (SQLite backend) |
| reportlab | mini_db (PDF reports) |

## Document storage

The `documents/` directory mirrors the DB table structure:

```
documents/Persons/<id>/     — national IDs, birth certificates
documents/Orphans/<id>/     — medical reports, case files
documents/Sponsorships/<id>/ — agreements
…
```

The `Documents` table's `path` column stores the relative path from
`documents/`, making it trivially joinable to any entity.
